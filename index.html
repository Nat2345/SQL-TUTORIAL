🚀 Introducción a Bases de Datos

¿Qué es una Base de Datos?

Una base de datos es una colección organizada de datos que se almacena y se accede electrónicamente desde un sistema informático. Los datos se organizan en tablas que contienen filas y columnas.

Tipos de Bases de Datos

  • Relacionales: Organiza datos en tablas con relaciones (MySQL, PostgreSQL, SQL Server)
  • NoSQL: Almacena datos de forma flexible (MongoDB, Cassandra)
  • Documentales: Basadas en documentos JSON
  • Gráficas: Optimizadas para relaciones complejas

Conceptos Básicos

Término Descripción
Tabla Estructura que contiene datos organizados en filas y columnas
Fila/Registro Un conjunto de datos relacionados en una tabla
Columna/Campo Una característica o atributo de los datos
Clave Primaria Identificador único para cada registro
Ejemplo de Tabla:
ID Nombre Email
1 Juan juan@email.com
2 María maria@email.com
💡 Consejo: Las bases de datos relacionales son las más usadas en aplicaciones web y comerciales.

🏗️ Diseño de Bases de Datos

Principios de Diseño

El buen diseño es fundamental para crear una base de datos eficiente, mantenible y escalable.

Pasos en el Diseño

  1. Análisis de requisitos: Entender qué datos necesitas
  2. Diseño conceptual: Identificar entidades y relaciones
  3. Diseño lógico: Crear el esquema de tablas
  4. Diseño físico: Optimizar para el rendimiento

Modelo Entidad-Relación (ER)

El modelo ER es una representación gráfica que identifica entidades, atributos y relaciones. Se usa para visualizar cómo se conectan los datos antes de convertirlos en tablas.

  • Entidades: Objetos del mundo real, como Estudiantes, Cursos o Instructores.
  • Atributos: Propiedades de las entidades, como nombre, email o fecha_inscripcion.
  • Relaciones: Conexiones entre entidades, por ejemplo, un estudiante inscrito en un curso.
  • Cardinalidad: Define cuántas entidades participan en la relación (1:1, 1:N, N:M).

Este modelo es importante para definir correctamente los requisitos y evitar estructuras de datos incorrectas.

Modelo Relacional

El modelo relacional convierte el diseño ER en tablas. Cada entidad se transforma en una tabla, cada atributo en una columna y cada relación en claves foráneas o tablas intermedias.

  • Tabla: Representa una entidad con columnas y filas.
  • Tupla: Cada fila de una tabla contiene un registro completo.
  • Dominio: Conjunto de valores válidos para cada columna.
  • Clave primaria: Identifica de forma única cada fila.
  • Clave foránea: Mantiene la integridad referencial entre tablas.

El modelo relacional permite diseñar consultas eficientes y aplicar reglas de normalización para eliminar redundancia.

Reglas de Diseño Importantes

  • Cada tabla debe representar una entidad única
  • Minimizar la redundancia de datos
  • Usar claves primarias para identificar registros únicos
  • Usar claves externas para relacionar tablas
  • Aplicar normalización
💡 Consejo: Usar herramientas como MySQL Workbench, ERDPlus o Lucidchart para visualizar el diseño.

➕ Crear BD y Tablas

Las cuatro imágenes muestran la creación de una base de datos y la anatomía de la primera tabla, con sus campos y restricciones.

Debajo se explica brevemente cómo aplicar esa estructura en SQL.

Crear una Base de Datos

CREATE DATABASE mi_base_datos;

Crear una Tabla Básica

CREATE TABLE estudiantes ( id INT PRIMARY KEY AUTO_INCREMENT, nombre VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE, edad INT, fecha_inscripcion DATE );

Tipos de Datos Comunes

Tipo Descripción Ejemplo
INT Números enteros 25, 100, -50
VARCHAR(n) Texto variable 'Juan', 'María'
DATE Fecha 2026-06-16
DECIMAL(p,s) Números decimales 99.99
BOOLEAN Verdadero/Falso TRUE, FALSE

Restricciones

CREATE TABLE productos ( id INT PRIMARY KEY AUTO_INCREMENT, nombre VARCHAR(100) NOT NULL, precio DECIMAL(10, 2) NOT NULL CHECK (precio > 0), stock INT DEFAULT 0, email VARCHAR(100) UNIQUE );
Restricciones Importantes:
  • PRIMARY KEY: Identifica únicamente cada registro
  • NOT NULL: El campo no puede estar vacío
  • UNIQUE: Todos los valores deben ser únicos
  • DEFAULT: Valor por defecto si no se especifica
  • CHECK: Valida condiciones
  • FOREIGN KEY: Referencia a otra tabla

✏️ Modificar y Eliminar Tablas

Modificar una Tabla (ALTER TABLE)

Agregar una Columna

ALTER TABLE estudiantes ADD COLUMN telefono VARCHAR(15);

Modificar una Columna

ALTER TABLE estudiantes MODIFY COLUMN nombre VARCHAR(150) NOT NULL;

Eliminar una Columna

ALTER TABLE estudiantes DROP COLUMN telefono;

Renombrar una Columna

ALTER TABLE estudiantes CHANGE COLUMN edad edad_actual INT;

Renombrar Tabla

RENAME TABLE estudiantes TO alumnos;

Eliminar una Tabla

DROP TABLE estudiantes;
⚠️ Cuidado: DROP TABLE elimina la tabla y todos sus datos permanentemente. Siempre haz una copia de seguridad.

Eliminar una BD Completa

DROP DATABASE mi_base_datos;
💡 Consejo: Para mayor seguridad, usa DROP TABLE IF EXISTS para evitar errores.

🔗 Foreign Key y Normalización

¿Qué es una Clave Foránea?

Una clave foránea es un campo en una tabla que hace referencia a la clave primaria de otra tabla, estableciendo una relación entre las tablas.

Crear Foreign Key

CREATE TABLE cursos ( id INT PRIMARY KEY AUTO_INCREMENT, nombre VARCHAR(100) NOT NULL, instructor_id INT, FOREIGN KEY (instructor_id) REFERENCES instructores(id) );

Normalización

La normalización es el proceso de organizar datos para reducir redundancia.

Primera Forma Normal (1NF)

  • Eliminar grupos repetitivos
  • Cada campo debe contener un solo valor

Segunda Forma Normal (2NF)

  • Cumplir con 1NF
  • Eliminar dependencias parciales

Tercera Forma Normal (3NF)

  • Cumplir con 2NF
  • Eliminar dependencias transitivas

📝 Insertar Registros

INSERT INTO - Sintaxis Básica

INSERT INTO tabla (columna1, columna2, columna3) VALUES (valor1, valor2, valor3);

Ejemplo 1: Insertar un Registro

INSERT INTO estudiantes (nombre, email, edad) VALUES ('Juan Pérez', 'juan@email.com', 20);

Ejemplo 2: Insertar sin Especificar Columnas

INSERT INTO estudiantes VALUES (NULL, 'María García', 'maria@email.com', 22, '2026-01-15');

Insertar Múltiples Registros

INSERT INTO estudiantes (nombre, email, edad) VALUES ('Ana López', 'ana@email.com', 21), ('Pedro Ruiz', 'pedro@email.com', 23), ('Laura Martín', 'laura@email.com', 20);

Insertar desde otra Tabla

INSERT INTO estudiantes_bak (nombre, email) SELECT nombre, email FROM estudiantes;
💡 Consejo: Los valores de texto deben ir entre comillas simples.
⚠️ Cuidado: Verifica que los tipos de datos coincidan con la definición de la tabla.

🗑️ Eliminar Registros

DELETE - Sintaxis Básica

DELETE FROM tabla WHERE condición;

Ejemplo 1: Eliminar un Registro Específico

DELETE FROM estudiantes WHERE id = 5;

Ejemplo 2: Eliminar Múltiples Registros

DELETE FROM estudiantes WHERE edad > 30;

Ejemplo 3: Eliminar todos los Registros

DELETE FROM estudiantes;

Diferencia entre DELETE y TRUNCATE

Aspecto DELETE TRUNCATE
Velocidad Más lento Más rápido
WHERE Se puede usar No se puede usar
Identidad Se preserva Se reinicia
Espacio No libera Libera espacio
⚠️ Cuidado: Siempre verifica la condición WHERE antes de ejecutar DELETE. Sin ella, eliminarás todos los registros.
💡 Consejo: Usa transacciones para mayor seguridad:
BEGIN; DELETE FROM estudiantes WHERE id = 5; COMMIT; -- o ROLLBACK para deshacer

🔄 Actualizar Registros

UPDATE - Sintaxis Básica

UPDATE tabla SET columna1 = valor1, columna2 = valor2 WHERE condición;

Ejemplo 1: Actualizar un Campo

UPDATE estudiantes SET edad = 21 WHERE id = 1;

Ejemplo 2: Actualizar Múltiples Campos

UPDATE estudiantes SET nombre = 'Juan Carlos', email = 'juan.carlos@email.com' WHERE id = 1;

Ejemplo 3: Actualizar con Cálculos

UPDATE productos SET precio = precio * 1.10 WHERE categoria = 'Electrónica';

Ejemplo 4: Actualizar Todos los Registros

UPDATE estudiantes SET estado = 'Activo';
⚠️ Cuidado: Siempre especifica una condición WHERE. Sin ella, actualizarás TODOS los registros.
💡 Consejo: Usa SELECT primero para verificar qué registros se actualizarán:
SELECT * FROM estudiantes WHERE id = 1; -- Verifica UPDATE estudiantes SET edad = 21 WHERE id = 1; -- Actualiza

🔍 Seleccionar Registros

SELECT - Sintaxis Básica

SELECT columna1, columna2 FROM tabla WHERE condición;

Ejemplo 1: Seleccionar Todas las Columnas

SELECT * FROM estudiantes;

Ejemplo 2: Seleccionar Columnas Específicas

SELECT nombre, email, edad FROM estudiantes;

Ejemplo 3: Seleccionar con Condición

SELECT * FROM estudiantes WHERE edad > 20;

Ejemplo 4: Seleccionar con Alias

SELECT nombre AS 'Nombre del Estudiante', edad AS 'Edad del Estudiante' FROM estudiantes;

JOINS - Combinar Tablas

INNER JOIN

SELECT e.nombre, c.nombre FROM estudiantes e INNER JOIN cursos c ON e.id = c.estudiante_id;

LEFT JOIN

SELECT e.nombre, c.nombre FROM estudiantes e LEFT JOIN cursos c ON e.id = c.estudiante_id;
💡 Consejo: Usa alias (AS) para hacer tus consultas más legibles.

⚙️ Operadores Lógicos y de Comparación

Operadores de Comparación

Operador Significado Ejemplo
= Igual a WHERE edad = 25
!= o <> No igual a WHERE edad != 25
> Mayor que WHERE edad > 25
< Menor que WHERE edad < 25
>= Mayor o igual que WHERE edad >= 25
<= Menor o igual que WHERE edad <= 25

Operadores Lógicos

Operador Significado Ejemplo
AND Todas las condiciones deben ser verdaderas WHERE edad > 20 AND edad < 30
OR Al menos una condición debe ser verdadera WHERE edad < 20 OR edad > 30
NOT Niega una condición WHERE NOT edad = 25

Otros Operadores Útiles

BETWEEN

SELECT * FROM estudiantes WHERE edad BETWEEN 20 AND 30;

IN

SELECT * FROM estudiantes WHERE ciudad IN ('Madrid', 'Barcelona', 'Valencia');

LIKE

SELECT * FROM estudiantes WHERE nombre LIKE 'J%'; -- Comienza con J

IS NULL / IS NOT NULL

SELECT * FROM estudiantes WHERE telefono IS NOT NULL;

Ejemplos Complejos

SELECT * FROM estudiantes WHERE (edad > 20 AND ciudad = 'Madrid') OR (edad < 18 AND ciudad = 'Barcelona') AND estado = 'Activo';
💡 Consejo: Usa paréntesis para clarificar la precedencia lógica de tus condiciones.

📊 Order By / Limit / Top

ORDER BY - Ordenar Resultados

SELECT * FROM estudiantes ORDER BY nombre ASC;

ORDER BY Descendente

SELECT * FROM estudiantes ORDER BY edad DESC;

ORDER BY por Múltiples Columnas

SELECT * FROM estudiantes ORDER BY ciudad ASC, edad DESC;

LIMIT - Limitar Resultados (MySQL, PostgreSQL)

SELECT * FROM estudiantes LIMIT 10; -- Primeros 10 registros SELECT * FROM estudiantes LIMIT 5 OFFSET 10; -- 5 registros a partir del 11

TOP - Limitar Resultados (SQL Server)

SELECT TOP 10 * FROM estudiantes; SELECT TOP 10 PERCENT * FROM estudiantes;

FETCH - Limitar Resultados (SQL Server, PostgreSQL)

SELECT * FROM estudiantes ORDER BY id OFFSET 10 ROWS FETCH NEXT 5 ROWS ONLY;

Paginación

-- Página 1: Primeros 10 registros SELECT * FROM estudiantes ORDER BY id LIMIT 10 OFFSET 0; -- Página 2: Registros del 11 al 20 SELECT * FROM estudiantes ORDER BY id LIMIT 10 OFFSET 10;
💡 Consejo: Usa ORDER BY antes de LIMIT para asegurar resultados consistentes.

📄 Funciones de Texto

Funciones de Texto Comunes

Función Descripción Ejemplo
UPPER() Convertir a mayúsculas UPPER('juan') = 'JUAN'
LOWER() Convertir a minúsculas LOWER('JUAN') = 'juan'
LENGTH() Obtener longitud del texto LENGTH('Juan') = 4
SUBSTRING() Extraer parte del texto SUBSTRING('Juan', 1, 2) = 'Ju'
TRIM() Eliminar espacios en blanco TRIM(' Juan ') = 'Juan'
REPLACE() Reemplazar texto REPLACE('Juan', 'u', 'a') = 'Jaan'
CONCAT() Concatenar textos CONCAT('Juan', ' ', 'Pérez')
REVERSE() Invertir texto REVERSE('Juan') = 'nauJ'

Ejemplos Prácticos

Convertir a Mayúsculas

SELECT nombre, UPPER(nombre) AS nombre_mayuscula FROM estudiantes;

Concatenar Campos

SELECT CONCAT(nombre, ' - ', email) AS informacion FROM estudiantes;

Extraer Parte del Texto

SELECT nombre, SUBSTRING(email, 1, 5) AS inicio_email FROM estudiantes;

Buscar y Reemplazar

SELECT REPLACE(descripcion, 'viejo', 'nuevo') AS descripcion_actualizada FROM productos;
💡 Consejo: Las funciones de texto son muy útiles para limpiar y normalizar datos.

🔢 Funciones Numéricas

Funciones Numéricas Comunes

Función Descripción Ejemplo
ABS() Valor absoluto ABS(-25) = 25
ROUND() Redondear ROUND(3.7) = 4
CEIL() Redondear hacia arriba CEIL(3.2) = 4
FLOOR() Redondear hacia abajo FLOOR(3.9) = 3
POWER() Potencia POWER(2, 3) = 8
SQRT() Raíz cuadrada SQRT(16) = 4
MOD() Módulo (resto) MOD(10, 3) = 1
RAND() Número aleatorio RAND() = 0.123...

Funciones de Agregación

Función Descripción Ejemplo
SUM() Suma total SUM(precio)
AVG() Promedio AVG(edad)
MIN() Valor mínimo MIN(precio)
MAX() Valor máximo MAX(edad)
COUNT() Contar registros COUNT(*)

Ejemplos Prácticos

Redondear Precios

SELECT nombre, ROUND(precio, 2) AS precio_redondeado FROM productos;

Calcular Suma Total

SELECT SUM(precio) AS total FROM productos;

Promedio de Edad

SELECT AVG(edad) AS edad_promedio FROM estudiantes;

Contar Registros

SELECT COUNT(*) AS total_estudiantes FROM estudiantes;

Precio Mínimo y Máximo

SELECT MIN(precio) AS precio_minimo, MAX(precio) AS precio_maximo FROM productos;

GROUP BY

SELECT ciudad, COUNT(*) AS total_estudiantes FROM estudiantes GROUP BY ciudad;
💡 Consejo: Usa COUNT(columna) para contar valores no nulos, o COUNT(*) para contar todas las filas.

🕐 Manejo de Fechas y Horas en SQL

Tipos de Datos de Fecha y Hora

Tipo Formato Rango
DATE YYYY-MM-DD 1000-01-01 a 9999-12-31
TIME HH:MM:SS -838:59:59 a 838:59:59
DATETIME YYYY-MM-DD HH:MM:SS Combinación de ambos
TIMESTAMP YYYY-MM-DD HH:MM:SS 1970-01-01 a 2038-01-19
YEAR YYYY 1901 a 2155

Funciones de Fecha y Hora

Función Descripción Ejemplo
NOW() Fecha y hora actual NOW() = 2026-06-16 14:30:00
CURDATE() Fecha actual CURDATE() = 2026-06-16
CURTIME() Hora actual CURTIME() = 14:30:00
DATE_ADD() Sumar días a una fecha DATE_ADD(NOW(), INTERVAL 7 DAY)
DATEDIFF() Diferencia entre fechas DATEDIFF(fecha1, fecha2)
DATE_FORMAT() Formatear fecha DATE_FORMAT(NOW(), '%d/%m/%Y')
YEAR() Extraer año YEAR('2026-06-16') = 2026
MONTH() Extraer mes MONTH('2026-06-16') = 6
DAY() Extraer día DAY('2026-06-16') = 16

Ejemplos Prácticos

Insertar Fecha Actual

INSERT INTO estudiantes (nombre, fecha_inscripcion) VALUES ('Juan', NOW());

Calcular Días de Diferencia

SELECT nombre, DATEDIFF(NOW(), fecha_inscripcion) AS dias_inscrito FROM estudiantes;

Sumar Días a una Fecha

SELECT DATE_ADD('2026-06-16', INTERVAL 30 DAY) AS fecha_futura;

Formatear Fecha

SELECT DATE_FORMAT(NOW(), '%d de %M de %Y') AS fecha_formateada;

Buscar Registros de Hoy

SELECT * FROM estudiantes WHERE DATE(fecha_inscripcion) = CURDATE();

Buscar Registros del Último Mes

SELECT * FROM estudiantes WHERE fecha_inscripcion >= DATE_SUB(NOW(), INTERVAL 1 MONTH);

Formatos de DATE_FORMAT Comunes

Formato Descripción
%d Día (01-31)
%m Mes (01-12)
%Y Año (4 dígitos)
%H Hora (00-23)
%i Minutos (00-59)
%s Segundos (00-59)
💡 Consejo: Siempre usa el formato YYYY-MM-DD para insertar fechas para evitar ambigüedades.
